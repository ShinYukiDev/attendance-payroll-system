# Quickstart: 勤怠入力画面

## 1. 前提

- .NET SDK（プロジェクトの TargetFramework に対応する版）をインストール済みであること
- リポジトリ直下を作業ディレクトリとする

## 2. 復元とビルド

```powershell
dotnet restore
dotnet build AtendancePayrollSystem.sln
```

## 3. テスト実行（ユニットテスト）

```powershell
dotnet test tests/AtendancePayrollSystem.Tests/AtendancePayrollSystem.Tests.csproj
```

## 4. アプリ起動

```powershell
dotnet run --project src/AtendancePayrollSystem/AtendancePayrollSystem.csproj
```

## 5. 勤怠入力機能の手動確認

### 一覧表示

- 社員ID、勤務日、出勤時刻、退勤時刻、休憩時間、実働時間が表示される。
- **実働時間の表示形式確認**: 実働時間が HH:mm 形式で表示されること（例: 9:30）

### 新規追加

- 有効な値で保存でき、一覧に追加される。
- **保存完了確認**: 保存が 3 分以内に完了し、一覧に新規行が反映されること（SC-001）

### バリデーション

- 社員IDが7桁数字以外の場合に保存拒否される。
- 退勤時刻 <= 出勤時刻の場合に保存拒否される。
- 休憩時間が不正な場合に保存拒否される。

### 更新

- 既存レコード更新後に実働時間が再計算される。
- **修正完了確認**: 修正が 2 分以内に完了し、一覧が更新されること（SC-002）

### 削除

- 削除確定でレコードが物理削除される。
- 削除確定でレコードが一覧から除外されることを確認

### 競合

- 同一レコード同時更新で後続保存が拒否され、再読込導線が表示される。

### 監査ログ

- 登録・更新・削除で監査ログが生成される。

## 6. ユーザー受け入れテスト（SC-004）

**対象者**: 初回利用者を含む最少 10 名

**テストシナリオ**: 以下を会議なしで実施（90% 以上が成功すれば合格）

1. **US1 (登録)**: 社員ID、勤務日、出勤時刻、退勤時刻、休憩時間を入力し保存。一覧に新規行が正しく追加されること。
2. **US2 (更新)**: 既存レコードを選択し、退勤時刻または休憩時間を変更し保存。実働時間が再計算されること。
3. **US3 (削除/一覧)**: 一覧画面を開いて全件が表示されること。対象行を削除して一覧から消えること。

**合格基準**: 【合格】全 3 シナリオを完了した会議者が総数の 90% 以上

## 7. 保持ポリシー確認

- 監査ログは3か月保持される。
- 日次の保持期限削除処理で超過ログが削除される。
- **マスキング確認**: 監査ログの TargetEmployeeIdMasked が先頭 2 桁 + 末尾 1 桁 (例: 12_7) でマスクされていること
